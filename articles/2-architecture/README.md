筆者がｽﾀｯｸﾁｬﾝをAIエージェント化するための作戦として、そのアーキテクチャと、考え方を解説します。

## 2.1. AIエージェントの稼働場所

AIエージェントを動かす場合、3つの要素があると思います。

- LLM推論の実行
- LLM推論を、AIエージェントとして動作させるフレームワークの実行

LLM推論には、クラウドサービスとして提供されている者から、ローカルでNPUなどを用いて動かすものまで、様々なものがあります。
CoreS3のようなMCU単体で、LLM推論を動かすことは、現状では難しいと思います。
M5Stackは、NPUを搭載モジュールも販売しており、それらを組み合わせることもできますが、LLM推論の性能には実行できるモデルに依存するため、そのような安価なNPUモジュールでは性能を期待できません。
普段コーディングエージェントでも利用しているのと同じ性能が得られればと思い、クラウドサービスを利用することにしました。

AIエージェントとして動かすには、LLM推論をAIエージェントのように利用するフレームワークが必要です。
OpenAIなどクラウドサービスには、AIエージェントとして稼働させる部分もAPIとして提供されていたりします。
今回、スマートホームのためのツールなどよりカスタマイズして動かしたいため、自宅サーバ内で動かすことにしました。
CoreS3のようなMCU単体でAIエージェントのフレームワークをゼロから実装することもできますが、気軽に作り込んでいくことは難しいと思いました。

あくまでエッジとなるｽﾀｯｸﾁｬﾝは、音声などのユーザインターフェイスだけを担当し、LLM推論とAIエージェントのフレームワークは、サーバ上で動かすアーキテクチャを考えました。

![AIエージェントアーキテクチャの概要](./img/llm_architecture.drawio.svg)

本書では、そのように作り込んでいく方法を解説していきます。
なお、本書の製作の再現には自宅サーバがなくても、PC上で実行できるようになっています。

## 2.2. 筆者のホームAIエージェントｽﾀｯｸﾁｬﾝのアーキテクチャ

筆者の自宅は、エアコン、TVなどのリモコンで操作できる家電が、既にREST APIで操作できるようになっています。
こちらの顛末については、技術書典17で頒布した同人誌「お家IoTリモコン全部作る」にまとめていますが、本書でも5章で簡単に紹介します。

サーバでAIエージェントフレームワークを動かすにあたり、それらをｽﾀｯｸﾁｬﾝから利用するアーキテクチャを考えました。

M5Stack CoreS3に使われているESP32-S3は、Wi-Fiを搭載しています。
Web技術を用いて、ESP32-S3と接続する方法として、WebSocketが使えます。
WebSocketは、サーバ、クライアントの構成ながら、双方向通信が可能なプロトコルです。
ｽﾀｯｸﾁｬﾝから、WebSocketを用いて接続し、AIエージェントとして必要な機能を自宅サーバ上で動かすアーキテクチャを考えました。
すると、多くの実装がサーバ上で完結できるため、実装が簡単になると考えました。

音声AIエージェントとして、対話できるようにするためには、以下の工程が必要です。

1. 動き出すきっかけの音声を聞き取る（ウェイクワード検出）
2. 人間の音声を録音する
3. 音声をテキストに変換する（音声認識）
4. テキストをAIエージェントで処理して、対話テキストを生成する
5. 対話テキストを音声に変換する（音声合成）
6. 音声を再生する

2、6、の工程は、CoreS3にて行います。
4はAIエージェントの処理なのでサーバ上で行います。

3の音声認識は、オープンソースを用いて自宅サーバ上で行うこともできますが、クラウドサービスとして高品質なものが提供されています。
3の音声を自宅サーバに送り、そこからクラウドサービスや、別の仕組みに繋いで音声認識を行うアーキテクチャを考えました。

- OpenAIのクラウドサービスのWhisper
- OpenAIが公開するWhisperのオープンソース実装であるWhisper.cpp
- Google Cloud Speech-to-Text

筆者は、仕事ではGoogle Cloudを使うことが多いため、Google Cloud Speech-to-Textを利用することにしました。
オープンソースWhisper.cppも良いのですが、NPUを搭載していない自宅サーバ（OrangPi 5）のCPUで動かすと遅いため、クラウドサービスを利用することにしました。
macOS上で実行する分には、GPUが使えるWhisper.cppが十分な性能で動いているため、そちらも選択肢としては良いと思います。

5の音声合成も、クラウドサービスとして高品質なものが提供されています。
一方ホビー用途では、無料で利用できる中品質と表している音声合成ソフトウェアであるVoicevoxがよく利用されます。
Voicevoxはずんだもんの声などでよく知られています。
かわいらしいアニメ声の音声も多く、私個人で使う分には良いように思いました。

これらを組み合わせると、以下のようなアーキテクチャーになります。

![実行フローのアーキテクチャ](./img/architecture.drawio.svg)

よって、これから以下のようなことを作り込んでいこうと思います。

- ｽﾀｯｸﾁｬﾝファームウェア
    - WakeupWord検出
    - WebSocket経由で、マイクデータを送ること
    - WebSocket経由で、スピーカー音を受け取って再生すること
    - WebSocket経由で、ｽﾀｯｸﾁｬﾝの動きを指示できること
- サーバ上のファームウェア
    - WebSocketサーバを立てること
    - 受け取ったマイクデータの音声認識
    - LLMエージェントの処理
    - LLMエージェントの応答の文字起こし（音声合成）
    - LLMエージェントから使える家電操作ツール
